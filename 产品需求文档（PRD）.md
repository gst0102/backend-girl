# 产品需求文档（PRD）  
## 项目名称：记录生活小程序  
### 版本：V1.0 | 更新日期：2026年5月

---

## 一、产品背景与目标

### 1.1 背景
- 当前用户可通过小程序记录「拉屎」「姨妈」等基础健康行为，并查看日历。
- 用户邀请好友可解锁更多记录类型、追番提醒、网盘资源等高阶功能。
- 运营日历已规划 12 周裂变活动，需在功能上与界面、进度、解锁机制打通。

### 1.2 核心目标
- 完成 **冷启动期 1000 核心用户** 拉新验证
- 通过 **邀请解锁机制** 驱动用户主动传播
- 为追番、情侣、家族等场景沉淀社交关系数据

## 三、核心机制详细设计

### 3.1 记录类型解锁体系

| 邀请人数 | 解锁内容               | 展示位置                      |
| -------- | ---------------------- | ----------------------------- |
| 0 人     | 💩 拉屎、🩸 姨妈         | 首页快捷记录区                |
| 1 人     | 😴 睡眠记录             | 展开区（??? 变为可点/可记录） |
| 3 人     | 💧 喝水提醒             | 同上                          |
| 8 人     | 📺 追番提醒（不含网盘） | 同上                          |
| 15 人    | 📺 追番预告             | 同上                          |
| 30 人    | 📺 网盘资源自动推送     | 追番中心 + 跑马灯             |

> 未解锁项显示为灰色 `???`，点击触发邀请弹窗。

### 3.2 邀请进度系统

- 用户邀请人数 = 已注册且通过其分享链接进入的好友数
- 跑马灯文案示例：
  
> 🎁 邀请30位好友，解锁定制影视剧网盘资源自动推送功能！当前进度：7/30，还差23人 → 点击邀请好友 →
  
- 邀请中心展示：
  - 当前邀请人数
  - 阶梯奖励（1人 / 3人 / 8人 / 15人 / 30人）
  - 已解锁 / 未解锁标签

### 3.3 弹窗提示逻辑

- 用户点击任意 `???` 记录项 → 弹出半屏锁定提示弹窗
- 弹窗内容：
  - 当前邀请进度
  - 解锁该功能所需人数
  - 按钮：「立即邀请好友」「稍后再说」

### 3.4 守护者 / 情侣 / 家族

| 关系类型 | 功能说明                                       |
| -------- | ---------------------------------------------- |
| 守护者   | 用户授权某人查看自己的健康日历（如姨妈、睡眠） |
| 情侣     | 互相授权后展示“拉屎同步率”，可生成同步率海报   |
| 家族     | 3人以上加入 → 家族健康树升级，解锁家庭药箱提醒 |

> 以上功能暂为展示型，后端需支持授权关系存储与日历访问控制。

## 四、页面与交互说明

### 4.1 首页（日历 + 快捷记录）

- 顶部：2026年5月日历，有记录日期红点标注
- 今日状态卡片：💩 已记录 / 🩸 第3天 / 😴 ???（未解锁）
- 快捷记录区：
  - 固定显示：💩 拉屎、🩸 姨妈
  - 下方折叠区（初始隐藏）：其他记录类型（如 😴、💧、📺 等），未解锁为 `???`
  - 展开/收起按钮支持动态切换

### 4.2 邀请中心

- 邀请进度条 + 邀请人数
- 阶梯奖励列表（已解锁 / 进行中 / 未解锁）
- 分享卡片预览 + 按钮：「邀请微信好友」「生成海报」

### 4.3 追番中心

- 仅对已解锁追番提醒的用户展示完整内容
- 展示“今日更新”番剧 + 网盘入口（仅对30人解锁用户开放）

### 4.4 我的页面

- 头像 + 邀请人数展示
- 入口：邀请好友、守护者、情侣、追番中心

## 六、非功能需求

### 6.1 数据统计需求
- 用户邀请人数
- 各解锁阶梯达成人数
- 排行榜（邀请人数 Top 10）

### 6.2 安全性
- 同一设备 / 同一微信ID防刷
- 授权关系不可越权查看他人数据

### 6.3 可扩展性
- 记录类型支持后台配置
- 解锁条件支持运营动态调整（人数 / 功能映射）
