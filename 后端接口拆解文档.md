# 后端接口拆解文档

> 目标：让大模型根据以下拆解，直接生成API接口代码和数据表结构
>
> ## 技术栈
>
> - 语言：Python 3.10
> - 框架：FastAPI
> - 数据库：PostgreSQL
> - ORM：SQLAlchemy
> - 架构：MVC
>   风格：RESTful API，统一响应格式

## 一、数据库表结构

### 1.1 用户表 `users`

| 字段名          | 类型        | 必填 | 默认值            | 说明                   |
| --------------- | ----------- | ---- | ----------------- | ---------------------- |
| id              | VARCHAR(32) | 是   | -                 | 用户唯一标识（自生成） |
| openid          | VARCHAR(64) | 是   | -                 | 微信openid             |
| nickname        | VARCHAR(50) | 否   | '用户'            | 昵称                   |
| avatar          | VARCHAR(20) | 否   | '👤'               | 头像emoji              |
| invite_count    | INT         | 是   | 0                 | 累计邀请人数           |
| continuous_days | INT         | 是   | 0                 | 连续记录天数           |
| last_record_at  | DATETIME    | 否   | NULL              | 最后一次记录时间       |
| created_at      | DATETIME    | 是   | CURRENT_TIMESTAMP | 注册时间               |
| updated_at      | DATETIME    | 是   | CURRENT_TIMESTAMP | 更新时间               |

### 1.2 邀请关系表 `invite_relations`

| 字段名         | 类型         | 必填 | 说明                     |
| -------------- | ------------ | ---- | ------------------------ |
| id             | INT          | 是   | 自增主键                 |

### 1.3 记录表 `records`

| 字段名       | 类型        | 必填 | 说明                                 |
| ------------ | ----------- | ---- | ------------------------------------ |

## 二、统一响应格式

### 2.1 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 2.2 失败响应

```json
{
  "code": 40001,
  "message": "参数错误：invitee_id不能为空",
  "data": null
}
```

### 2.3 错误码枚举

| 错误码 | 说明                     |
| ------ | ------------------------ |
| 0      | 成功                     |
| 40001  | 参数错误                 |
| 40002  | 用户不存在               |
| 40003  | 记录已存在（今日已记录） |
| 40004  | 功能未解锁               |
| 40005  | 邀请关系已存在           |
| 40006  | 不能邀请自己             |
| 40007  | 守护关系已存在           |
| 40008  | 情侣关系已存在           |
| 40009  | 番剧不存在               |
| 40010  | 已订阅该番剧             |
| 40101  | 未登录                   |
| 40301  | 无权限访问               |
| 40401  | 资源不存在               |
| 42901  | 操作过于频繁             |
| 50001  | 服务器内部错误           |

## 三、接口列表

### 3.1 用户登录/注册

**接口：** `POST /api/user/login`

**入参：**

```json
{
  "code": "微信登录code",
  "inviter_id": "可选，邀请人ID"
}
```

**出参：**

```json
{
  "code": 0,
  "data": {
    "user_id": "xxx",
    "is_new": true,
    "invite_count": 0,
    "unlocked_features": ["poop", "period"]
  }
}
```

**业务规则：**

1. 通过code获取openid
2. 查询openid是否存在
3. 不存在则创建新用户
4. 若存在inviter_id且为新用户，创建邀请关系
5. 触发invite_success事件
6. 返回用户信息

**异常处理：**

- code无效 → 返回40001
- inviter_id用户不存在 → 忽略，正常注册

### 3.3 创建记录

**接口：** `POST /api/record/create`

**入参：**

```json
{
  "user_id": "必填",
  "record_type": "poop",
  "record_date": "2026-05-18",
  "record_value": null
}
```

**出参：**

```json
{
  "code": 0,
  "data": {
    "record_id": 12345,
    "continuous_days": 6,
    "badges_earned": [
      { "id": "badge_002", "name": "连续记录7天", "icon": "🔥" }
    ]
  }
}
```

**业务规则：**

1. 校验record_type是否已解锁
2. 同一天同类型不允许重复记录
3. 记录创建后更新user.last_record_at
4. 计算连续记录天数
5. 检查是否触发徽章（连续7/14/30天）
6. 返回新获得的徽章列表

**异常处理：**

- record_type未解锁 → 返回40004
- 今日已记录同类型 → 返回40003
- user_id不存在 → 返回40002

### 3.6 创建邀请关系

**接口：** `POST /api/invite/create`

**入参：**

```json
{
  "inviter_id": "必填",
  "invitee_openid": "必填",
  "invitee_device": "可选"
}
```

**出参：**

```json
{
  "code": 0,
  "data": {
    "invite_count": 8,
    "new_unlocked": ["anime_remind"]
  }
}
```

**业务规则：**

1. 校验inviter_id和invitee_openid不是同一个人
2. 校验invitee_openid未被其他用户邀请过
3. 创建邀请关系
4. inviter.invite_count + 1
5. 检查是否达到解锁阈值（1/3/8/15/30）
6. 批量更新unlocked_features
7. 返回新解锁的功能列表

**异常处理：**

- 邀请自己 → 返回40006
- 已被邀请过 → 返回40005

## 四、定时任务

### 4.1 每日任务（00:00执行）

| 任务名称         | 逻辑                                     |
| ---------------- | ---------------------------------------- |
| 更新连续记录天数 | 检查用户昨日是否有记录，有则+1，无则归零 |
| 更新情侣绑定天数 | couple_relations表bind_days + 1          |
| 更新家族等级     | 重新计算level = floor(成员数/3) + 1      |
| 清理过期邀请     | 删除超过30天未确认的邀请关系             |

### 4.2 每周任务（周一00:00执行）

| 任务名称       | 逻辑                         |
| -------------- | ---------------------------- |
| 更新排行榜缓存 | 重新计算invite_count Top 100 |
| 推送周报       | 向有记录用户推送周报         |

### 4.3 每小时任务

| 任务名称       | 逻辑                                    |
| -------------- | --------------------------------------- |
| 处理待发送推送 | 从push_logs读取status=0的记录，执行发送 |

## 五、事件触发（内部调用）

### 5.1 邀请成功事件

**触发时机：** 新用户通过邀请链接注册

**处理逻辑：**

```
1. 更新inviter.invite_count + 1
2. 检查解锁阈值
3. 如有新解锁功能，更新user_features
4. 插入reward_logs
5. 创建推送记录（inviter）
```

### 5.2 记录创建事件

**触发时机：** 用户创建新记录

**处理逻辑：**

```
1. 更新user.last_record_at
2. 检查连续记录天数变化
3. 检查徽章触发（7/14/30天）
4. 如有新徽章，更新user_badges
5. 更新情侣同步率缓存
```
