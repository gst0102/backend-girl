## 后端新增接口建议：上传公众号二维码

### 上传公众号二维码
| 项目 | 内容 |
|------|------|
| 方法 | POST |
| 路径 | /api/admin/upload/qrcode |
| 认证 | Bearer Token (Admin) |
| Content-Type | multipart/form-data |

**请求体：**
```text
file: File (必填，图片文件，建议支持 png/jpg/jpeg/webp/gif，大小限制建议 2MB 或 5MB)
```

**响应体：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "https://cdn.example.com/uploads/qrcode/official_account_xxx.png"
  }
}
```

**错误码建议：**
| 错误码 | 含义 |
|--------|------|
| 40001 | 文件为空 / 文件类型不支持 / 文件过大 |
| 40101 | 未登录或 Token 无效 |
| 40301 | 非管理员 |
| 50001 | 上传或存储失败 |

**前端保存流程：**
1. 管理后台先调用 `/api/admin/upload/qrcode` 上传图片。
2. 后端返回 `data.url`。
3. 前端把该 URL 写入 `official_account.qrcode_url`。
4. 管理员点击“保存预留配置”时，继续调用现有 `/api/admin/reserve` 保存：
```json
{
  "official_account": {
    "enabled": true,
    "name": "公众号",
    "qrcode_url": "https://cdn.example.com/uploads/qrcode/official_account_xxx.png",
    "description": "测试描述"
  }
}
```

> 兼容说明：当前前端也兼容返回 `data.qrcode_url`，但推荐后端统一返回 `data.url`。
