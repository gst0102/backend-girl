## 管理后台接口报错排查结果

排查时间：2026-05-26

前端请求可以打到后端，后端返回 HTTP 200，但部分接口业务响应为 `code: 50001`。因此这些不是前端路由或网络断开问题，而是后端接口实现或数据库结构问题。

| 数据看板 | `GET /api/admin/stats/trend?days=7` | `code: 50001` | `Invalid format string` |
| 用户管理 | `GET /api/admin/users?page=1&page_size=20` | `code: 50001` | `type object 'UserFeature' has no attribute 'unlocked'` |

### 后端需要处理

1. 修复 `stats/overview` 中 SQLAlchemy 聚合写法，不能在 `func.sum(...)` 后直接 `.where(...)`。
2. 修复 `stats/trend` 日期格式化字符串。
3. 修复用户列表的 `UserFeature.unlocked` 字段引用，当前数据表是 `user_features.feature_key`，没有 `unlocked` 字段。
4. 给数据库 `ad_config` 补齐字段：`test_unit_id`、`ab_test_enabled`、`ab_test_ratio` 等接口文档中声明的字段，或调整 ORM 不再查询不存在的字段。
5. 创建 `marquee_config` 表，或让接口使用实际存在的表名。

## 后端新增接口建议：上传 Banner 图片

### 上传 Banner 图片
| 项目 | 内容 |
|------|------|
| 方法 | POST |
| 路径 | `/api/admin/upload/banner` |
| 认证 | Bearer Token (Admin) |
| Content-Type | multipart/form-data |

**请求体：**
```text
file: File (必填，图片文件，建议支持 png/jpg/jpeg/webp/gif，大小限制建议 5MB)
```

**响应体：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "https://cdn.example.com/uploads/banner/banner_xxx.png"
  }
}
```

**错误码建议：**
| 错误码 | 含义 |
|--------|------|
| 40001 | 文件为空 / 文件类型不支持 / 文件过大 |
| 40101 | 未登录或 Token 无效 |
| 40301 | 非管理员 |
| 50001 | 上传或存储失败 |

**前端保存流程：**
1. 管理后台先调用 `/api/admin/upload/banner` 上传图片。
2. 后端返回 `data.url`。
3. 前端把该 URL 写入 Banner 表单的 `image_url`。
4. 管理员点击保存时，继续调用现有 `/api/admin/banners` 或 `/api/admin/banners/{id}` 保存 Banner。

> 兼容说明：当前前端也兼容返回 `data.image_url`，但推荐后端统一返回 `data.url`。
